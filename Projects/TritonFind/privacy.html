TritonFind Privacy Policy

Your privacy is important to us. Here is how we protect it.

Data Collection

TritonFind collects and uses data from a variety of sources. These are the specfic places we use.

Data From Facebook

TritonFind can access the following information from users, courtesy of the Messenger Bot API Platform:

  • First Name
  • Last Name
  • Gender
  • Profile Picture
  • locale (e.g. "US-en")
  • Timezone

The only information TritonFind actually accesses a user's First Name, Last Name, and Profile Picture.

TritonFind uses a persons first name for two reasons:

  1. Personalize Messages
  2. Share with another user

When we share a person's first name with another person, it is done with permission of said user. If a user finds a lost item, they choose to send a "conversation invite" to the owner. This is the explicit permission a user gives to share their first name with the owner. Also, if one receives a "conversation invite," if they accept the invite, that is the explicit permission they give to TritonFind to share their first name with the other user.

TritonFind also accesses a person's last name. For now, it is not used, but may be used in the future.

TritonFind uses a person's Facebook profile picture. If a user is in a conversation with another user through TritionFind (in order to communicate on an items whereabouts), they have the option to send the other user their profile picture (in order to show them what they look like). This is only done with explicit person from the user, they must purposefully choose the option to send the other person their own profile picture.

TritonFind does not store any information that has been gathered by the Messenger User Profile API. So, we do not store anyone's first name, last name, or profile picture. It is only accessed when needed, never stored.

Data From Users

TritonFind receives much of its data from users themselves. Users can send TritonFind pictures, barcodes, texts, photos, or even their location. Here are all of sources of data TritonFind receives, and what TritonFind does with them.

Messenger Messages

TritonFind communicates all through Messenger. All messages can be viewed in TritonFind's Facebook Page Messages. However, the TritonFind team will only access these messages under the following circumstances:

  • LiveChat with users
  • Debugging TritonFind
  • Investigate Reports
  • Comply with Law Enforcement
LiveChat

Users have the option to join in a live chat with the TritonFind team. Once a user requests for a LiveChat (to discuss TritonFind functionality, ask questions, file reports, etc.), the TritonFind team may access this person's messages on TritonFind.

Debugging

Sometimes, TritonFind does not work the way it should be. So, our team may access the TritonFind Page Messages to see what went wrong. However, we will only access our own Messaging threads, and not view other users' messages. That will only be done with explicit permission from said users.

Reports

Users may break the TritonFind Terms of Service during conversation with other users. So, if a user files a report against another user, the TritonFind team may access the messages threads of all parties affiliated with the report filed. Our team will keep any information seen as confidential.

Law Enforcement

If any local law enforcement agencies request specified threads for their own investigations, TritonFind will comply.

Pictures And Barcodes

TritonFind's main functionality rests on its ability to read and save barcodes from pictures sent by users. The TritonFind team strives to keep this information safe and secure.

TritonFind stores all barcode information in a database. This information will never be used by the TritonFind team, for any purposes. The data is never accessed for any reason by our team. Item information can be deleted by a user at any time. We also reccommed that users delete photos sent to TritonFind, for extra security.

Pictures sent to TritonFind are not stored, either. They can be accessed through the TritonFind Facebook Page Messages, but will only be accessed by the TritonFind team in the instances labelled above.

Location

TritonFind gives the option of a user sharing their location with another user. This is done when the two users are in a conversation, trying to return a lost item. A user must explicitly send their location to TritonFind, and TritonFind will relay the location to the user other. Locations are never stored on TritonFind's side.

Logging

TritonFind logs much of its activity, through Heroku's logging system. This is done in order to debug and mantain TritonFind's many features.
This can, however, cause some privacy issues. some message information may be logged during TritonFind activity. However, the TritonFind team will only access these logs only when necessary to debug and mantain TritonFind. These logs are stored securely, and automatically get deleted after a certain amount of time.

Updates

When TritonFind's Privacy Policy changes in a significant way, we will alert our users through a TritonFind Facebook Page post. We will also utilize the TritonFind Announcement system to let our users know on Messenger.

Contact

If you feel that the TritonFind team is not abiding by this privacy policy, please send us a message through TritionFind and request to LiveChat. Alternatively, you can find our contact information here.
